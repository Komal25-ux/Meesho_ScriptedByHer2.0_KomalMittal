## 3. How You'll Be Evaluated

Your prototype will be evaluated based on:
- **Problem understanding**
- **Innovation**
- **Functionality**
- **Impact**
- **Design**
- **Overall completeness**

> ⚠️ **Important:** Incomplete submissions may not be considered for evaluation.

---

## 4. Prototype Evaluation Criteria (Detailed)

### 01 — Working Prototype
**Focus:** Functional & bug-free MVP
- Core features must be **implemented**, not mocked or hardcoded.
- Judges should be able to **run and test** the prototype independently.

### 02 — Code Quality & Architecture
**Focus:** Clean & modular code
- Well-structured architecture with proper **readme.md** documentation.
- Test coverage and coding best practices must be followed.

### 03 — Usability & User Experience
**Focus:** Intuitive & user-friendly
- Well-designed for the target user.
- Smooth navigation and interaction flows.

### 04 — Completeness
**Focus:** End-to-end coverage
- All key features built — complete user flows.
- **Not** just a partial demo or a single screen.

| Criterion | What Judges Ask |

## 6. Submission Requirements & Deliverables

> Make your submission **complete, transparent, and review-ready!**

### 01 — Project Core
- **Description:** Clear overview of the problem, proposed solution, and AI model integration details.
- **Presentation:** Pitch deck or document outlining the overall business concept and technology.

### 02 — Live Deployment
- **Live Demo:** Accessible URL of the deployed application or a hosted running instance.
- **Interactivity:** Ensure judges can test core interactions, workflows, and view model responses directly.

### 03 — Code & Setup
- **Source Code:** Link to your public repository (e.g. GitHub/GitLab) with complete project code.
- **README Guide:** Detailed setup file with clear instructions on how to run and verify the codebase locally.

### 04 — Open-Source Attribution
For every library, framework, or tool used, declare:
- **Name & version**
- **License type** (MIT, Apache 2.0, GPL v3, etc.)
- **Role in your build** (direct integration, modified code, conceptual inspiration, etc.)
- **Source link** to the original project
